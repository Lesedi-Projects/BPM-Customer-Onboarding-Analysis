# Process Scoping and Analysis

This folder contains the BPTrends scoping diagram for the 
customer application review and onboarding process.

## What is a BPTrends Scoping Diagram?
A BPTrends scoping diagram is a tool used in business process 
management to map everything that influences a process before 
analysing it in detail. It organises the process environment 
into six categories, helping analysts understand the full 
context — not just the steps in the process itself.

## The Six Categories
- **Inputs** are what triggers the process to start
- **Guides** are policies, rules, and regulations that govern 
  how the process must be performed
- **Outputs** are what the process produces when complete
- **Enablers** are the tools, systems, and technology used
- **Organisational units** are the roles and departments involved
- **Process** are the high-level steps in the centre

## What This Diagram Shows
The scoping diagram maps the customer onboarding process across 
all six categories identifying the stakeholders, systems, 
policies, and outputs that define the boundaries and context 
of the process before detailed analysis begins.
